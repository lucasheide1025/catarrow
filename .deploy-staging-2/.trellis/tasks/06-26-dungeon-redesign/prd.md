# PRD — 地下城系統全面重設計

## 背景

現有地下城模式僅有「一樓層 → AB 路線 → 條件戰鬥」，缺乏探索感與打寶感。
本次重設計將地下城升級為完整的多樓層探索系統，並支援 1~8 人組隊。

---

## 核心設計確認

| 項目 | 規格 |
|------|------|
| 地圖類型 | 固定設計（hardcode data），後台控制開/關 |
| 最大人數 | 8 人（1 房主 + 7 隊友） |
| 入場券 | 只扣房主；每日打卡取得（不累積）+ 道具型（可累積） |
| 戰鬥系統 | HP 制（沿用現有 archer HP 計算） |
| 進度儲存 | 只保留房主記憶，下次可重新招募繼續 |
| 商人房貨幣 | 金幣（house gold） |
| 符文定位 | 地下城專屬裝備，補充現有 RPG 裝備做不到的功能 |

---

## 功能需求

### 1. 地下城資料結構（dungeonData.js）

每個地下城包含：
- `id`, `name`（如「幽冥地窖」「水晶洞窟」）
- `floors`：陣列，每層有 `rooms` 陣列
- `rooms`：每間有 `type`、`position`（x,y）、`connections`（連通哪些房間）
- Room types：`monster` `elite` `boss` `chest` `trap` `merchant` `rest` `teleport` `event` `stairs`

### 2. 探索地圖 UI（DungeonMap.jsx）

- SVG 繪製房間節點 + 連線
- 顏色區分房間類型；已探索/未探索/當前位置
- 組隊時所有人看同一張地圖
- 隊長提議移動方向 → 全員 30 秒投票 → 平手由隊長定奪

### 3. 房間事件系統

| 類型 | 行為 |
|------|------|
| monster | 進入戰鬥（1~3 隻怪） |
| elite | 精英怪，必有寶箱獎勵 |
| boss | Boss 戰，掉落專屬寶物 + 符文 |
| chest | 直接開寶箱，依樓層掉落素材/金幣 |
| trap | 全體扣血（10~25% 最大 HP） |
| merchant | 商店，用金幣購買道具/符文 |
| rest | 全體回復 30% HP |
| teleport | 隨機傳送至同層已探索房間 |
| event | 特殊文字事件（buff / debuff / 選擇題） |
| stairs | 前往下一層 |

### 4. 地下城戰鬥

回合順序：前衛出手 → 後衛（補血 or 輸出-30%）→ 貓貓陪練 → 怪物反擊

- 複數怪物（1~3）：玩家自選攻擊目標 or 隨機
- 怪物反擊：優先打前衛；特殊攻擊可打後衛（傷害 ×1.5）
- 全體送出 → 5 秒倒數 → 跑動畫結果
- 逃脫：確認視窗，保留已入背包的寶物

### 5. 符文系統（runeData.js）

| 符文 | 效果 |
|------|------|
| 復活符文（低階） | 陣亡時復活一次 +20% HP |
| 多重復活符文（低階） | 陣亡時復活兩次 +20% HP |
| 強攻符文 | 攻擊力 +15% |
| 守護符文 | 受到傷害 -15% |
| 貓靈符文 | 貓貓陪練傷害 +25% |

裝備位置：獨立於現有 RPG 裝備，地下城進場前選擇最多 1 顆符文。

### 6. 寶物與圖鑑

- 每個地下城有專屬寶物清單（3~5 種）
- Boss 掉落 1~3 項，普通寶箱掉落通用素材
- 新增「地下城圖鑑」：收集到的寶物解鎖展示
- 寶物可作為裝備升級素材或賣給商人換金幣

### 7. 進度持久化（Firestore）

```
members/{memberId}/dungeonRun
  dungeonId, floorIndex, roomStates (explored/cleared),
  currentRoomId, partySnapshot, collectedLoot, status
```

### 8. 結算通知

- 每間房結束立即顯示當次獎勵
- 通關或房主主動離開 → 顯示完整結算
- 透過訊息中心發彈窗給已離場隊友

### 9. 後台管理

- AdminApp 新增「地下城管理」tab
- 控制每個地下城的 enabled 開關
