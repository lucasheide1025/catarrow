# 後台提示音與約課結帳連動修正

## Goal

讓教練能清楚聽見且辨識報到、新預約與下一小時預約提示，並確保審核中心結帳一定回寫正確 booking，不再出現會計已有帳但約課仍可結帳。

## Requirements

1. 報到、新預約、下一小時預約使用三種可明顯辨識的旋律。
2. 三種提示音的感知音量需顯著高於目前版本，並在教練閱讀前依既有週期持續播放。
3. 閱讀／點擊相應提示後停止該類提示音。
4. 審核中心結帳時，即使 checkin 沒有 bookingId、同日有多筆預約，仍要以 checkin 建立時間與 booking 時段選出最合理的一筆。
5. fallback 必須同時考慮 confirmed 與「已完成但尚未標記 billingRecordId」的 booking。
6. 找不到 booking 時不得假裝連動成功；帳務已建立的情況需保留 billing ID 供重試，避免重複帳。
7. 成功連動後，線上約課立即顯示已結帳，不再提供結帳按鈕。
8. 正式學生在線上約課後台結帳前，必須已有當日 checkin 且 `classEnded:true`；尚未按下課時不可啟動結帳。
9. 正式學生若審核中心尚未結帳，才允許從線上約課結帳；成功後同步 checkin 並使審核中心直接完成。
10. 若審核中心已結帳但 booking 漏寫 billingRecordId，線上約課需補連動既有 billing ID，不得再建立帳。
11. 臨時訪客、公開訪客與 guest 帳號不受報到／下課門檻限制，可直接結帳。

## Acceptance Criteria

- [ ] 三種提示音音量明顯提高且旋律不同。
- [ ] 未閱讀時每 12 秒持續提醒；點擊後停止對應聲音。
- [ ] 同日單筆預約的審核中心結帳正確寫入 booking.billingRecordId。
- [ ] 同日多筆預約依 checkin 時間選到最近／涵蓋時段的 booking。
- [ ] completed 但無 billingRecordId 的 booking 仍可由審核中心補連動。
- [ ] fallback 無匹配時回報錯誤，不關閉成已成功。
- [ ] 正式學生未按下課時，線上約課無法結帳。
- [ ] 正式學生在線上約課結帳後，審核中心不再要求結帳。
- [ ] 訪客預約可不經報到直接結帳。
- [ ] 正式版建置與 Firestore 規則檢查通過。
