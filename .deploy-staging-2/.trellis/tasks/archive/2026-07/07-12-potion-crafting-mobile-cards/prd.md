# 藥水工坊手機小卡與戰鬥測試修正

## Goal

將藥水製作介面重構為適合手機快速閱讀與批量製作的分類小卡，並確保後台新版戰鬥 UI 測試頁可正常編譯進入。

## Requirements

- 攜帶型、投擲型、討伐型保留主頁籤，頁籤內再依用途分組並顯示短標題。
- 攜帶型分為回復、輸出、防護、預備；投擲型分為直接傷害、弱化、支援控制、預備；討伐型分為傷害與戰術。
- 手機版從第一個可用寬度起固定兩欄小卡；較寬畫面可增加為三欄，不退回單欄長列表。
- 每張卡直接顯示圖片、名稱、效果、庫存、單次產量、依目前批量模式換算後的材料／金幣與製作按鈕。
- 卡片高度與圖片區域穩定，長名稱、效果與材料不可互相覆蓋或撐動同列布局。
- 製作 1 次、5 次、最大數量控制與目前金幣保持容易操作；按鈕至少 44px 高。
- 預備道具清楚標示尚未開放使用，但仍可製作。
- 修正 `AdminBattleTest.jsx` React hooks 重複宣告編譯錯誤；只使用模組 import，不再從全域 `React` 重複解構。
- 不覆蓋 Claude 同時修改的預約後台、戰鬥測試或 `AdminApp` 其他內容。

## Acceptance Criteria

- [ ] 320px 寬度下藥水卡維持兩欄，文字、材料、產量與按鈕皆可閱讀及點擊。
- [ ] 每個主頁籤內的品項出現在正確用途分組，沒有遺漏或重複。
- [ ] 切換 1／5／最大數量後，每張卡的需求與產量同步更新，材料不足狀態正確。
- [ ] 圖集載入成功時顯示對應圖片，失敗時維持 emoji fallback。
- [ ] `AdminBattleTest.jsx` 不存在 hook 重複宣告，後台戰鬥版式入口能通過編譯。
- [ ] 自動測試與 production build 通過。

## Out of Scope

- 不改變任何藥水效果、配方、產量或戰鬥規則。
- 不將後台測試版式套用到正式戰鬥畫面。
