## 為什麼要改
現在的戰鬥畫面是全站最舊的一塊：大量 inline style、沒接上後來的設計系統、單調、且**不同戰鬥模式各長一樣（拼湊感）**。目標：**一套版式跨所有戰鬥模式**（單人打怪 / 組隊 / 地下城 / 世界王），直式手機、傾斜視角、四角分工、中間留白給動畫。

---

## 版式規格（定案）

**畫面方向**：直式手機（portrait）。傾斜 arena 視角＝天空→地面漸層 + 透視地格暗示景深；實裝時直接鋪各族專屬背景圖。

**分區（重要，落地時照這個擺）**：
```
┌ 頂部資訊列：所處地城 · 難度 · 第幾層 ───────────┐
│  戰鬥訊息(左, 與怪物切齊)      怪物(右上)         │
│                                                  │
│            ← 中間一大片留白 = 動畫舞台 →          │
│  ┌ 隊友格框 ┐                                     │
│  └ 玩家卡框 ┘                          功能選單    │
└──────────────────────────────────────────────────┘
```

| 區域 | 內容 | 備註 |
|------|------|------|
| **頂部資訊列** | `🏚️ 地城名 · 難度 · 第 N / 總 層` | 依實際地城/樓層動態帶入；單人打怪可只顯示「打怪」或該場資訊 |
| **左上 戰鬥訊息** | 最近 1–2 行 | 位置與右上怪物切齊（不貼頂）；只留關鍵訊息不塞滿 |
| **右上 怪物** | `MonsterSVG` 立繪 + 血條 + 名稱等級 | **沿用原本 MonsterSVG 的方框卡樣式**（使用者確認用原素材即可） |
| **左下 兩個大框**（用一個 `team-cluster` flex column 容器堆疊，左對齊、寬度一致） | 上框＝**隊友格**、下框＝**玩家卡** | 兩框務必用容器包好正確堆疊，不可各自 absolute 亂擺互相卡住 |
| **右下 功能選單** | 計分 / 藥水 / 重置 / 放棄 | 計分＝金色主按鈕(最顯眼)；放棄＝紅色放最下(避免誤觸) |
| **中間** | 留白 | 給箭矢、傷害數字、擊殺特效、貓貓演出 |

**箭數指示**：右下選單上方一排小圓點＝本回合箭數進度（打了幾箭）。

## 多人處理（1～8 人，關鍵）

- **每個人在自己手機上只計自己的分** → 「你」永遠是**放大的玩家卡**（左下），其他隊友壓成小格子。
- **隊友格**：疊在玩家卡**正上方**（左下一整叢，4+3 格 wrap），每格只給四件事：
  1. **HP 細條**（誰快死一眼看到）
  2. **前/後排框色**：前排=橘(`--front #ffb454`)、後排=藍(`--rear #7dd3fc`)
  3. **右上狀態點**：綠勾✓=本回合已送出、金沙漏…=等待(閃爍)、✕=陣亡(灰掉)
- **點任一隊友格 → 跳詳細資訊卡**：頭像、前/後排、HP、ATK/DEF、貓咪夥伴、本場貢獻傷害。
- 單人模式：隊友格框隱藏，只留玩家卡框。**同一版式四模式共用**，差別只在隊友格框有沒有東西。

## 三種畫面狀態
1. **乾淨戰鬥**（預設）：四角 + 中間留白。
2. **計分中**：點「計分」→ 背景變暗聚焦、中央由下升起輸入區逐箭輸入 → 「送出」回乾淨畫面。
   - **數字模式**＝跳數字鍵盤(X/10/9…1/M)；**靶面模式**＝中央改顯示靶面(此版式不變，靶面計分邏輯照舊)。
3. **隊友詳情**：點隊友格 → 置中彈卡，點背景/關閉收起。

# 🔁 交接：前後衛模型改版 + 戰鬥模擬器現況（2026-07-12，Claude → FREEBUFF / 下一個 session）

## A. 定案：前後衛新模型（組隊 + 組隊地下城 統一）
- **開場全員前衛**，**取消等待室「選前後衛」步驟**、**取消前衛固定 4 人上限**。
- **第一次陣亡 → 自動復活符回 50% HP → 轉後衛**（此後 heal/assist、不直接輸出）。
- **後衛再死＝真死**。
- **全員都被打倒過（存活前衛歸零）＝全體判輸**（後衛打不死怪）。
- UI 橘/藍框＝**狀態顯示**（橘＝未倒過的前衛、藍＝倒過的後衛/待救援），不是選擇。

## E. 戰鬥模擬器 `src/components/admin/AdminBattleTest.jsx` 現況（⚠️ 全部本機、未 commit）
