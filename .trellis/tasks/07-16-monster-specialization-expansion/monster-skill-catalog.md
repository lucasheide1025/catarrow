# Monster skill catalog

## 1. 共用資料契約

每個技能定義必須包含：`id`、顯示名稱、適用 Tier、效果積木、倍率／回合數、預告文字、破解條件、完整破解效果、部分破解效果、動畫 key、音效 key、reduced-motion key。所有模式只傳入戰鬥上下文，由同一 resolver 產生結果；UI 不自行計算傷害、異常、護盾或隨機結果。

技能資料在戰鬥／房間建立時鎖定版本與排程，單人以 battle ID、組隊以 room ID + round、地下城以 run ID + room ID、世界王以 attempt ID + round 作冪等鍵。重連只重播已儲存結果，不重新抽選、重算或重複扣血。

- 此表是招牌技能的安全基準，不包含世界王第 2／4 回合固定強攻。

- 同一招牌技能最多包含「一次直接傷害＋一個狀態」或「無傷害＋兩個輕量狀態」，禁止疊三種懲罰。
- T1～T2 不出現反射、淨化、多回合再生、不可見預告或超過 1 回合的控制。

## 3. 射箭破解判定

### 單人

以當回合有效箭的品質分數計算：10 分=1.0、9 分=0.9、8 分=0.75、7 分=0.55、6 分以下=0.3、脫靶=0。`breakRatio = 實得品質分數 ÷ 當回合最大品質分數`。

### 組隊

只計入本回合已合法送出結果的玩家，固定依前／中／後排與 member ID 排序彙總；`breakRatio = 全隊實得品質分數總和 ÷ 全隊最大品質分數總和`。貓咪傷害不列入破解值。

### 結果級距

| breakRatio | 結果 | 技能處理 |
|---:|---|---|
| ≥85% | 完整破解 | 取消傷害與附加狀態；播放短破招演出 |
| 70%～84.99% | 大幅削弱 | 傷害×0.35；取消附加狀態 |
| 50%～69.99% | 部分削弱 | 傷害×0.70；狀態強度或回合數減半，至少 1 回合 |
| <50% | 未破解 | 完整效果 |

純護盾／回復／姿態技能使用同一級距：完整破解取消；大幅削弱保留 25%；部分削弱保留 60%；未破解完整生效。

## 5. 招牌技能效果積木

| Primitive | 用途 | 限制 |
|---|---|---|
| `direct_hit` | 單次直接傷害 | 使用 Tier／怪物類型基準倍率 |
| `multi_hit` | 2～3 段攻擊 | 總倍率不得高於同級 `direct_hit` +10%；只播放一次整體演出 |
| `pierce_def` | 無視部分防禦 | 一般 10%、小王 15%、大王 20% 上限 |
| `pierce_shield` | 無視部分護盾 | 一般 10%、小王 20%、大王 30% 上限 |
| `self_shield` | 取得護盾 | 最大 HP 5%／7%／10%，不得與回復同招共存 |
| `damage_reduction` | 暫時減傷 | 10%／15%／20%，最長 1 回合 |
| `atk_down` | 降低玩家輸出 | 5%／8%／10%，最長依 Tier 異常基準 |
| `def_down` | 降低玩家防禦 | 5%／8%／10%，最長依 Tier 異常基準 |
| `heal_down` | 降低玩家受治療 | 10%／15%／20%，不得阻止治療至 0 |
| `accuracy_mark` | 射擊挑戰標記 | 指定 9 分以上、特定環數或連續品質，不封鎖射擊 |
| `delayed_burst` | 下一回合延遲攻擊 | 必須明確倒數並可破解；傷害上限為基準×1.2 |
| `lifesteal` | 將本次實際傷害轉為自身HP | 僅一般／地下城怪可用，上限為實傷10%；世界王與安分寶箱禁用 |
| `bind_mark` | 束縛型射擊標記 | 不鎖按鈕、不跳過玩家回合；只轉譯為ATK／DEF降低或品質挑戰 |
| `target_scope` | `single`／`party` 目標選擇 | 單人固定single；組隊party走團隊破解，single走被指定玩家破解 |
| `cleanse_one` | 移除一個怪物負面狀態 | 僅 T3～T6；同回合不得再自我回復 |
| `phase_aura` | 大王 70%／40% HP 被動 | 每階段只觸發一次；只改一項數值或規則 |

## 7. 類型配置規則

- 一般 0.8：1 個共用技能＋偏控制／輔助的招牌技能；招牌基準傷害再×0.9。
- 一般 1.0：1 個共用技能＋保留既有角色語意的招牌技能。
- 一般 1.2：1 個共用技能＋偏直接攻擊的招牌技能；不得同時配置高倍率與強異常。
- 小王 A：2 個共用技能＋進攻型招牌技能，使用小王 A 的高 ATK 定位。
- 小王 B：2 個共用技能＋防守／控制型招牌技能，使用小王 B 的高 HP／DEF 定位。
- 大王：2 個共用技能＋一個招牌技能＋70%／40% HP 各一次的階段被動。第二階段只能強化已預告的技能，不新增無預告秒殺。

## 8. 回合排程

- 一般怪：R2 招牌、R4 共用，之後招牌／共用交替；每回合最多一個主動技能。
- 小王：R2 招牌、R4 共用 A、R6 共用 B，之後循環。
- 大王：R2 招牌、R4 共用 A、R6 強化招牌、R8 共用 B；HP 70%／40% 階段被動獨立觸發但只播放短轉場。
- 貓咪在怪物技能前行動；任一貓咪擊倒怪物時，取消當回合怪物技能。
- reduced motion 下移除震屏、粒子尾跡與鏡頭推進，只保留 150ms 以內的亮框、圖示和數值變化。

## 9. 唯一演出與音色資料

- 每個招牌技能固定 `id=sig_{monsterId}`、`animationKey=anim_sig_{monsterId}`、`telegraphKey=tele_sig_{monsterId}`，確保252招皆可獨立追蹤與測試。
- `paletteKey` 由族系與Tier組合，例如 `ghost_t3`；同族維持一致語言，Tier提高只增加裝甲細節與亮度，不增加閃爍頻率。
- `soundKey` 由主要 primitive 與族系組合，例如 `mountain_direct_hit`、`exam_accuracy_mark`；技能可共用短音色檔但每筆資料必須明確指定，不允許執行期猜測。
- `hapticKey` 只有 `none/light/medium/heavy`；預告一律none，普通命中light，小王／大王破招medium，只有世界王第4回合未破解可heavy。使用者關閉震動後不得呼叫Vibration API。
- 所有招牌技能需有一句兒童可理解的預告及一句破解成功文字；不得使用「笨、失敗者、被處罰」等羞辱語句。
