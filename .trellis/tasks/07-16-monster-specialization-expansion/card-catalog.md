# Card catalog

## 收集與升星

- 252 隻怪各一張同 ID 卡。舊卡持有、星級、重複數與裝備位置不變。
- 一般怪升星重複卡需求固定1／2／3／4／5，升至6星合計15張；小王、大王採相同升星需求，不另外提高。
- 首次擊敗王怪保證卡；其後小王20%且第5次保底，大王10%且第8次保底。一般怪沿用各戰鬥模式卡率。
- 王卡是不同戰術選擇，不保證高於同 Tier 一般卡；世界王卡系統維持獨立。

## 卡片效果公式

延續現有「族系決定屬性、星級決定數值」模型，避免252張卡各自產生難以維護的被動 resolver。

| 族系 | 主屬性 |
|---|---|
| ghost／workplace | HP |
| mountain／temple | ATK |
| insect／exam | DEF |
| treasure | 持有者在HP／ATK／DEF中選一項，首次選擇免費，之後沿用現有重選規則 |

`baseStatByTier = [1,2,3,4,5,6]`，卡片實際加成=`baseStatByTier × stars × roleFactor`，最後四捨五入：一般0.8=`0.9`、一般1.0=`1.0`、一般1.2=`1.1`、小王A=`1.05`、小王B=`1.05`、大王=`1.1`。王卡差異以配置彈性而非碾壓數值呈現。

## 介面與效能

- 族系→Tier分層，一次只渲染6張；可篩選一般／小王／大王、持有、可升級，可搜尋名稱。
- 未持有卡只顯示灰階剪影、名稱輪廓、遭遇類型與來源；取得後才顯示完整技能與故事。
- 圖片懶載入，分組切換中止失效請求；圖片應提供手機尺寸版本。不得一次載入252張原圖。
- 裝備、卸下、升星與市集既有操作保留；所有按鈕需有文字標籤、鍵盤焦點與44px觸控區。
- 360／390／430px 下每列2張，詳細資料使用單一底部面板；不得用橫向卡片牆。
