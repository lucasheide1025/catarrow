# 預約狀態 Email 通知

## Goal

預約建立確認、改期與取消後，自動寄送一次通知到該預約的有效 `contactEmail`，並另外通知教練。

## Requirements

- 由後端監聽 `bookings` 寫入並建立受信任的 `mail` 文件。
- 建立、改期、取消使用不同且清楚的主旨與內容，包含日期、時間及方案資訊。
- 教練通知包含學生姓名、Email、日期、時間、來源與事件類型。
- 三位教練收件者：`chobitsgl1@gmail.com`、`beluga0109@gmail.com`、`broudes@gmail.com`。
- 教練通知使用單一 mail 文件：`broudes@gmail.com` 放 `to`，其餘兩位放 `bcc`，避免彼此顯示完整地址並減少重複任務。
- 使用穩定事件鍵避免函式重試、文件後續更新或擴充功能回寫造成重複寄送。
- 學生與教練通知使用各自的穩定事件鍵；其中一位投遞失敗不得造成另一封重複建立。
- 無 Email 或格式無效時安全跳過，不影響預約操作。
- 完成／報到／結帳等非本次狀態變化不寄信。

## Acceptance Criteria

- [x] 新預約 confirmed 只寄一封確認信。
- [x] 改期只寄一封改期信，清楚列出新時段。
- [x] 取消只寄一封取消信。
- [x] 每個建立、改期、取消事件，教練各收到一封對應通知。
- [x] 重播相同 Firestore 事件不會建立第二封 mail。
- [x] 非目標更新不寄信。
