# 預約 Email 全中文化

## Goal

讓學生與教練收到的所有預約 Email 都是自然、清楚的繁體中文，不暴露 `general`、`walk_in` 等內部代碼，也不再使用生硬的資料庫欄位格式。

## Requirements

- 重寫全部八種範本：學生確認、改期、取消、課前一天、兩週未預約，以及教練確認、改期、取消。
- 教練信件改成完整中文敘述，不再以「事件：新預約」作為第一行。
- 將方案代碼轉為中文：`general` 單人一般、`discount` 兒童／學生／敬老、`own_equipment` 自備器材；未知值顯示「未指定方案」，不得直接輸出內部代碼。
- 將來源代碼轉為中文：`online_public` 訪客線上預約、`online` 學生線上約課、`phone` 教練代為預約、`walk_in` 教練現場新增；未知值顯示「其他方式」。
- 日期改為台灣中文年月日，時間加入上午／下午語意，人數加上「人」。
- Email 無效或缺少時顯示「未提供」，不得顯示 `null`、空字串或內部值。
- 所有模板仍只允許既有安全 token，收件者與 mail 欄位不得由模板控制。
- 正式 Firestore 內已保存的舊版預設範本必須安全升級；不能只更新程式預設值而繼續寄出舊文案。
- 保留後台自由編輯、預覽與測試寄信能力。
- 所有預約通知 mail 文件明確指定寄件者為 `貓小隊室內射箭場 <broudes@gmail.com>`，讓收件匣顯示場館名稱；實際 SMTP 帳號仍維持 `broudes@gmail.com`。

## Acceptance Criteria

- [x] 八種預設信件皆為自然繁體中文，主旨與內文語意符合各事件。
- [x] 已知方案／來源完整中文化，未知代碼不外洩。
- [x] 建立、改期、取消、課前一天與兩週提醒的日期、時間、人數格式正確。
- [x] 舊版預設範本在部署後由版本化正規化邏輯使用新版文案；後台載入後也顯示新版。
- [x] 使用者自訂且已標記新版的範本不會在每次載入時被覆蓋。
- [x] 即時通知、課前一天、兩週提醒與測試信均使用「貓小隊室內射箭場」作為寄件者顯示名稱。
- [x] Functions 測試、前台 build、語法與 diff check 通過。

## Out of Scope

- 不改收件者、通知開關、寄送排程或防重寄規則。
- 不在 Email 內加入 HTML、圖片或追蹤像素。
