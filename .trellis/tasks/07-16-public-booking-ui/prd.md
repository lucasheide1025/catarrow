# 訪客預約介面與課表優化

## Goal

讓訪客在手機上快速理解課程方案、可預約日期、完整時數與登入／註冊流程，同時隱藏不必要的內部容量資訊並降低操作負擔。

## Requirements

- 訪客版時段只顯示「可預約」「已額滿」「不可預約／公休」等狀態，不顯示新生、舊生、目前人數或剩餘名額數字。
- 容量判斷仍沿用 `slotState()` 與 Firestore slot counts；只改訪客呈現，不弱化送出交易的容量驗證。
- 日期範圍為台灣今天起至一個月內，不產生過去日期，也不提供向前導覽。
- 課程／時段選擇與確認摘要完整顯示 `開始時間－結束時間`、`1／2／3 小時`、方案、人數及總價，不要求使用者自行相減。
- 未登入手機畫面同時提供「立即預約」主行動與清楚可見的「會員登入／會員中心」次行動。
- 重排訪客預約、註冊、登入、問卷、確認與成功畫面，採手機優先響應式版面。
- 保留 Email＋密碼、Google 登入、電話補填、會員中心、取消與改期功能。
- 使用語意化元件、清楚 focus state、至少 44px 點擊區與不依賴 hover 的手機互動。

## Acceptance Criteria

- [x] 訪客時段不洩漏新／舊生或容量數字，且 disabled 狀態與既有判斷一致。
- [x] 日期最早為今天、最晚不超過一個月後，時區使用 Asia/Taipei。
- [x] 1、2、3 小時方案在時段、摘要、登入前確認、會員中心與改期畫面皆清楚顯示完整時段與時數。
- [x] 360px 寬手機可直接看到登入入口，主要流程無水平溢出。
- [x] 訪客可完整走完選課、登入／註冊、送出、查詢、取消與改期。
- [x] Production build 與相關測試通過（全站既有射手靶面測試 2 項失敗，與本任務無關；其餘 76 項及 production build 通過）。
