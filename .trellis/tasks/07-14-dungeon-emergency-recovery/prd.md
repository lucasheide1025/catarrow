# 地下城緊急復原

## Goal

恢復地下城戰鬥的可進入性，並使它的角色數值與一般遊戲戰鬥一致地納入已裝備卡片、貓咪與貓咪羈絆等加成。

## Requirements

- 修復進入地下城戰鬥時的 `ReferenceError: Cannot access 'myWbFrame' before initialization`。
- 保留地下城既有的角色與怪物圖片、靶面與回合輸入流程；未達完整相容性的共用戰鬥畫面不得取代它。
- 地下城建立的玩家戰鬥資料須保留並套用目前遊戲中的已裝備卡片加成。
- 地下城建立的玩家戰鬥資料須保留並套用已裝備貓咪與貓咪羈絆相關的數值／戰鬥效果。
- 核對地下城單人戰鬥與冒險單人打怪的設定接入；地下城專屬的房主鎖定箭數與靶面設定不被個人戰鬥設定覆蓋。
- 不得破壞現有單人與組隊地下城流程，亦不得覆寫工作區中與本復原無關的未提交修改。

## Acceptance Criteria

- [ ] 開啟單人地下城與組隊地下城戰鬥時不再出現 `myWbFrame` ReferenceError 或白屏。
- [ ] 單人地下城進入戰鬥後，角色與怪物圖片均正常顯示。
- [ ] 進入地下城後，玩家的 HP、ATK、DEF 與一般遊戲戰鬥使用相同來源的卡片與貓咪加成。
- [ ] 貓咪羈絆／技能在地下城戰鬥中依既有一般戰鬥規則可用，或明確維持既有產品規格的等效效果。
- [ ] 已記錄哪些單人打怪設定可共用，及哪些因地下城房間契約而刻意不同。
- [ ] 相關靜態檢查與建置通過，並完成可重現的單人及組隊地下城進入戰鬥驗證。

## Notes

- 回報環境：本機開發站 `localhost:3000`；錯誤元件為 `DungeonBattleRoom`。
- 初步診斷：`DungeonBattleRoom` 在宣告 `myWbFrame` 的 `useMemo` 前便於新版 `BattleScreen` 分支讀取它，觸發 JavaScript 暫時性死區。
- 既有工作區含大量未提交變更；修復須維持最小且可回復的範圍。
