# 射手表現首頁

## 需求

建立學生端第一版射手表現頁，資料只讀取新的 `shootingSessions` 摘要文件與 `gamePerformances`，呈現實際射箭表現與遊戲戰績，兩者不可混算。

## 約束

- 僅讀取目前登入者的有限筆數資料，使用一次性查詢；不可新增常駐監聽。
- 首頁不可讀取 `shootingSessions/*/ends`；逐箭精準統計留待詳細頁。
- 無資料時需提供清楚的空狀態。
- 近期 30／60／90 箭在只有摘要資料的第一階段，允許以完整 Session 近似，但須在 UI 明示。
- 新入口放在「我的戰績」中，不改變既有頁面的可見行為。

## 驗收條件

- [ ] 可從「我的戰績」開啟射手表現頁。
- [ ] 顯示近期平均、X 率、M 率、箭數、最佳單場平均。
- [ ] 顯示戰鬥數、勝率、總傷害及最高單場傷害，且不影響射箭指標。
- [ ] 資料讀取有上限、無 `onSnapshot`、不讀 `ends` 子集合。
